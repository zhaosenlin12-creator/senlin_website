# 森林老师个人站「动态轨道剧场」升级设计规格

**日期：** 2026-06-18  
**视觉基准：** `docs/design-references/2026-06-18-orbital-theatre-selected.png`

## 目标

把现有文字偏密、模块边界偏硬的内容区升级为以作品图像和真实教学照片为主的个人数字展厅。访问者应在短时间内理解赵森林老师兼具编程教育、应用开发、竞赛指导和持续内容输出能力，并能直接进入真实应用体验。

## 视觉方向

沿用现有近黑背景、青色强调、冷白标题和中英混排体系，不重做品牌。应用区采用「动态轨道剧场」：两条错位横向轨道承载电影感应用海报；教学相册采用更轻盈的椭圆轨道，照片围绕当前焦点运动。布局保留大面积留白和一个明确视觉焦点，避免每个区块都被独立大卡片包裹。

## 内容原则

1. 页面先展示成果，再补充最少量解释。
2. 应用卡只保留应用名、短分类和「进入体验」动作，不显示长段描述或三条卖点。
3. 内容区主标题改为「作品现场」，说明限制为一句短句。
4. 教学相册卡片默认只显示标题和分类，详细说明仅在放大层中出现。
5. 删除 `Stage Feature`、`Poster Rail` 等无必要的内部式命名，避免脏文案。

## 应用资产

为下列八个应用生成统一的 16:10 高质量横向海报，最终文件保存到 `public/media/apps/`：

1. `python-adventure.webp`：少年在科技森林中的 Python 闯关场景。
2. `classroom-feedback.webp`：课堂反馈、成长曲线与家校连接场景。
3. `typing-fun.webp`：未来键盘训练与节奏反馈场景。
4. `ai-research.webp`：学生与 AI 研究网络、知识节点互动场景。
5. `code-research.webp`：代码阅读、调试和逻辑拆解场景。
6. `interactive-classroom.webp`：教师与学生围绕实时生成内容互动。
7. `model-training.webp`：样本、训练和结果可视化实验场景。
8. `simulation.webp`：机器人与科学仿真环境。

所有图片使用相同的电影感 3D/写实插画语言、深蓝黑底、青色轮廓光，并允许少量琥珀暖光区分重点。图片内不生成文字、Logo、水印或 UI 标签，所有可访问文本由 HTML 提供。

## 应用轨道交互

1. 桌面端显示两条反向、错位、缓慢移动的无缝轨道。
2. 鼠标或键盘焦点进入某条轨道时，该轨道暂停；当前卡片抬升、轻微透视并强化边缘光。
3. 卡片整体为真实 `<a>`，统一 `target="_blank"` 与 `rel="noopener noreferrer"`。
4. 新标签打开后原主页保持滚动位置和交互状态；这是无法控制第三方页面时最可靠的返回方式。
5. 移动端取消自动轨道，改为横向滑动、滚动吸附和明确的「进入体验」按钮；浏览器返回手势不会被横向组件拦截。
6. `prefers-reduced-motion` 下关闭自动移动和大幅变换。

## 教学相册交互

1. 桌面端照片沿椭圆轨道排布，当前照片位于前景，其余照片根据与当前索引的距离调整缩放、透明度和层级。
2. 支持自动轮换、悬停暂停、左右拖拽、滚轮/方向键、上一个/下一个按钮和点击打开大图。
3. 放大层写入浏览器历史状态；浏览器返回或移动端返回手势先关闭大图，不离开主页。
4. 移动端使用横向滚动吸附，保留按钮与点击放大，不强行模拟桌面 3D。
5. 所有控制具备可读 `aria-label`，焦点样式清晰。

## 页面结构

1. 保留 Hero、About、数据、专业、荣誉、认证和联系区。
2. About 的两段长文压缩为一句定位和三项能力标签。
3. `media` 区顺序调整为：短标题、应用双轨、教学相册、精选抖音入口。
4. 抖音区只保留两张精选内容和主页入口，不重复解释用户应该如何浏览。
5. 使用不对称列宽、跨列图片与留白形成节奏，减少连续等宽矩形。

## 响应式与性能

1. 图片使用 `loading="lazy"`、`decoding="async"`，首张重点图可正常优先加载。
2. 轨道复制仅用于视觉无缝循环，复制项从键盘和辅助技术导航中排除。
3. 桌面、平板、手机分别验证 1440×1024、1024×768、390×844。
4. 不新增重型依赖，继续使用 React、Framer Motion、Lucide 和 CSS。

## 验收标准

1. 八张应用卡均使用专属高质量图片并链接到配置的真实地址。
2. 应用轨道桌面自动移动且悬停/聚焦暂停，移动端自然横滑。
3. 相册具备拖拽、按钮、键盘、暂停和放大交互；浏览器返回能关闭放大层。
4. 内容区不再出现原有长段应用说明和重复标签。
5. 外链均在新标签打开，并使用 `noopener noreferrer`。
6. 自动化测试、生产构建、桌面与移动端视觉验收全部通过。
