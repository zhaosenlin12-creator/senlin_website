# Bruno Simon Reference Notes

Source: https://bruno-simon.com/
Capture: `bruno-simon-home.png`, `bruno-simon-capture.json`

## What It Is

Bruno Simon is the playful interaction reference. The captured page is a canvas/3D world with almost no DOM text. Its memorability comes from controlling an object through a world rather than reading a conventional portfolio.

## Transferable Ideas

- A personal website can have a toy-like signature interaction that people remember.
- The environment can become a metaphor for the person's work. Bruno uses a drivable world; Senlin can use a learning universe or classroom control field.
- Visual simplicity helps the interaction remain legible: one grid, one object, one clear interaction model.
- A small interactive object can communicate technical ability faster than a paragraph.

## What Not To Copy

- Do not make Senlin's entire website a game. It would reduce institutional trust and slow down access to important information.
- Do not require keyboard/game controls to read the site.
- Do not use a heavy 3D scene on every device.

## Senlin Translation

- Add one contained "interactive learning universe" section, not a whole-site game.
- Let users drag or hover through 5-6 nodes: Python, C++, AI, Robotics, Competition, Projects.
- Each node reveals one concise teaching philosophy, project link, or achievement.
- Make it optional and responsive: on mobile it becomes a touch-friendly orbit/segmented explorer.
